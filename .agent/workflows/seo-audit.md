# /seo-audit

Use this workflow to review search and accessibility basics for `playground-chatgpt-app`.

## Repo profile

- Stack: `React application, TypeScript`
- Stack key: `react-app`
- Progression source: `PROJECT_PROGRESSION.md`

## Applicability

This stack has public web surfaces, so SEO review is first-class.

## Audit scope

- Review route metadata, titles, descriptions, canonicals, and robots behavior.
- Check heading hierarchy, link text, image alt coverage, and obvious crawl blockers.
- Inspect sitemap, static export, or route generation behavior if the stack supports it.

## Output

- list findings by severity
- include file paths and concrete fixes
- update progression only if the audit changes project phase or blockers

## Do not do

- Do not rewrite progression docs into an audit report
- Do not claim production SEO gains without evidence
