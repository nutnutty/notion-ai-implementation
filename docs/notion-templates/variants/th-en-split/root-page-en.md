## Purpose

Reusable root-page template for split-language mode, aligned to the same operational contract as all other pages.

## When to use

- Creating or refreshing any root page in the 01..07 set.
- Standardizing operational pages before rollout.

<!-- REQUIRED:How to use -->
## How to use

1. Define page objective and accountable owner.
2. Keep active priorities and next actions at top.
3. Link relevant database views.
4. Update weekly outcomes and decisions.

- Records from core databases and review loop

- Actionable status with traceability
- Explicit ownership and deadlines
